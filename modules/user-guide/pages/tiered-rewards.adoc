= Tiered rewards
:page-aliases: usage:reference:audience-controlled-rewards.adoc

Tiered rewards let you send one notification to a broad audience while giving different rewards to different player segments. The goal is to meet each player where they are in the game — an offer that's exciting for a casual player might not move a big spender at all.

A bundle can have any number of tiers, each targeting a different audience.

== How tiered rewards work

When a player claims a reward, Teak checks each tier from top to bottom and applies the first one whose audience includes that player. That player receives only that tier's reward — evaluation stops there. Players who don't match any tier receive the *Default Tier* reward.

The Default Tier is always present. It acts as a catch-all for any player not covered by a more specific tier above it.

.With this Tiered Reward, players who have spent at least $100 in the past 30 days receive 50,000 coins, players lapsed for at least 7 days receive 20,000 coins, and everyone else receives 1,000 coins.

Four players claiming this reward:

* *Bob* — spent $120 in the past 30 days, hasn't played for 10 days → matches Tier 1 first → 50,000 coins. (Bob is also in the lapsed audience, but evaluation stopped at Tier 1.)

* *Frank* — no spend, hasn't played for 7 days → doesn't match Tier 1, matches Tier 2 (lapsed) → 20,000 coins

== Set up tiers

For basic bundle setup, see xref:page$rewards.adoc[Reward bundles]. To add tiers:

. Open the bundle editor and check *Use Reward Tiers*
. Click *Insert Tier* to add a custom tier above the Default Tier
. Select the audience for that tier from the dropdown
. Add the rewards players in that audience should receive
. Repeat for each additional tier
. Use the reorder buttons to set evaluation priority — the topmost tier is evaluated first

[IMPORTANT]
.Order your tiers from most specific to least specific
====
A player who qualifies for a narrower audience (e.g., "Big spenders") should be matched before they can fall into a broader one (e.g., "Spenders"). If "Spenders" sits above "Big spenders," every big spender matches the first tier and never reaches the higher-value reward.
====

== Choose when tiers are evaluated

Teak determines which tier a player belongs to at a specific point in time. This matters because audience membership can change between when a notification is sent and when a player opens it.

=== Send-time evaluation (notifications and emails)

The tier is locked when the notification is sent. Each player's reward is determined by their audience membership at that moment — before anyone opens the notification.

Use send-time evaluation when the notification copy references the reward amount. The amount in the copy and the amount delivered will always match, even if the player's audience membership changes before they tap.

.With this Tiered Reward Bundle, players who have spent at least $100 in the past 30 days will get 50,000 coins, and all other players will get 1,000 coins.
image::rewards/payers_get_more.jpg[]

=== Click-time evaluation (links)

The tier is re-evaluated when the player clicks the link. The reward is based on audience membership at the moment of the click.

Use click-time evaluation when the most current audience data should determine the reward — for example, if a player's segment might change between send and click, and you want the reward to reflect their current state.

=== Editing a bundle after sending

If you modify a bundle after a notification has been sent, the original send-time evaluations are invalidated. Players who haven't clicked yet will have their tier re-evaluated at click time using their current audience membership — the same behavior as a link reward.

NOTE: This means any templated notification copy will no longer match the reward players actually receive. Always double-check your tier configuration before sending.

This also provides a safety valve: if you discover a misconfiguration after sending — for example, the default tier accidentally gives 1,000,000 coins instead of 1,000 — you can edit the bundle immediately to limit further damage.

== Use template tags with tiers

When a player can see the specific reward waiting for them — rather than a vague promise — they're more likely to tap. Use template tags to show each player their exact reward amount in the notification copy. This is especially valuable with tiers, where VIP players can see the higher amount they'll actually receive.

To insert a reward tag:

. In the notification message editor, click the template tag button (the variable icon in the toolbar)
. Under the *REWARDS* section, select the reward item whose amount you want to display

.The REWARDS section in the template tag dropdown. Each item listed resolves to the quantity that player will receive based on their tier.

After selecting the Tiered Reward as the Reward Bundle for a notification, Teak adds available reward tags to the dropdown automatically.

.The "Coins Amount" template tag resolves to the number of coins each player will receive based on their tier.

== Common patterns

*Reward loyalty.* Put your highest-spend or most-engaged audience in the top tier with a premium reward. The Default Tier gives a standard reward to everyone else.

*Re-engage lapsed players.* Add a tier for players inactive for 7+ days with a larger reward to bring them back. Players still active receive a smaller reward from a lower-priority tier or the Default Tier.

*Seasonal events.* Create a time-limited tier with an event-specific audience and a higher reward. Archive the bundle after the event ends and restore it for the next one.

*Combine with randomization.* Enable both *Use Reward Tiers* and *Use Random Rewards* on the same bundle. Teak evaluates the tier first, then randomizes within that tier's reward set. Each tier can have independent odds — giving VIPs better minimum and maximum outcomes than casual players. See xref:page$rewards.adoc#randomized[randomized rewards] for setup details.
