= Bundle list filters

As your reward bundle library grows, the filter controls on the Rewards page help you find what you're looking for quickly.

== Filter by bundle type

Click *Filter* on the Reward Bundles list to narrow the view by bundle characteristics.

.The Reward Bundles list with Sort, Filter, and Search controls. Bundles show a summary of their reward structure inline.
image::rewards/1_bundle_list.jpg[]

[cols="1,2",options="header"]
|===
|Filter option |What it shows

|All
|Every active (non-archived) bundle

|Audience controlled
|Bundles with at least one custom tier in addition to the Default Tier

|Not audience controlled
|Bundles with only a Default Tier — no custom audience targeting

|Randomized
|Bundles using odds-based reward selection

|Not randomized
|Bundles that deliver all rewards on every claim

|Show archived
|Includes archived bundles in the list
|===

Filters reflect the current state of each bundle. A bundle with both tiers and randomization enabled appears in both "Audience controlled" and "Randomized" results.

== Search and sort

Use the *Search* box to find bundles by name. Use *Sort* to order the list by creation date or name.

== When to use filters

*Looking for bundles that target specific player segments?* Select *Audience controlled* to see all bundles with custom tiers.

*Want bundles that give a fixed, predictable reward?* Select *Not randomized*.

*Need to reuse a seasonal bundle you archived?* Enable *Show archived*, find the bundle, and click *Restore*.

== Understanding the list view

Each bundle shows a summary of its reward structure inline:

* *Fixed bundle:* "Claimants receive all of: [reward list]" — every player gets the same rewards
* *Randomized bundle:* "Claimants receive one of: [reward list with chances]" — Teak picks one reward option per claim; the average claim value is shown below the options
* *Tiered bundle:* "Claimants receive rewards by audience" — each tier's audience and reward list is shown separately

The icon to the left of each bundle name also signals its type: a gift icon for fixed bundles, a shuffle icon for randomized bundles.

== Archive and restore bundles

Archiving removes a bundle from the active list without deleting it. See xref:page$rewards.adoc#bundle-creation[Reward bundles → Archive a bundle] for steps.
